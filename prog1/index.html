Första gången

A) Installera Visual Studio Code. (Om du inte redan gjort det...)

B) Installera Node.js. Välj Windows Installer (.msi), 64-bit. Då filen laddats ner så starta den (dubbelklicka).

C) Installera TypeScript-kompilatorn genom att:

  1. Öppna VS Code.
  2. Välj View - Terminal för att visa terminalfönstret. 
  3. Skriv där kommandot (avsluta med Enter): npm install -g typescript
  4. Kontrollera att installationen görs, det tar ca 10 sekunder.

D) Skriv också detta kommando i terminalfönstret (avsluta med Enter):  Set-ExecutionPolicy -ExecutionPolicy Unrestricted -Scope CurrentUser

E) Installera tillägget Live Server. Klicka på den gröna Install-knappen.

F) Man kan bestämma vilken webbläsare som ska öppnas med inställningen: File - Preferences - Settings:

Alla moderna webbläsare, till exempel Chrome och Firefox funkar bra.

 
G) Installera också tillägget Blade Runner.
 

H) En del tillägg kräver att VS Code startas om, så gör det.

Varje gång

A) Inifrån VS Code så väljer du File - Open Folder och väljer där den mapp som du vill jobba med.

B) Om du vill göra ett nytt program så ta först en kopia av någon mapp och ge den ett beskrivande namn:

  • Markera den mapp du vill kopiera (till exempel balder-app).
  • Kopiera med Ctrl+C.
  • Klistra in med Ctrl+V.
  • Byt till ett mer beskrivande namn på mappen.

C) Markera den mapp du vill jobba med och välj sedan Select Folder. (Du behöver alltså inte klicka dig in i själva mappen.)

D) Visa filen app.ts. Stäng gärna fönstret med alla filer så får du bättre plats med koden.

E) Tryck på Go Live-ikonen (längst ner i fönstret). Då öppnas en webbläsarflik som sedan uppdateras varje gång din fil sparas.

F) Skriv kod i app.ts - spara - och se resultatet i webbläsaren.

Tips:

  • Visa editorn till vänster och webbläsaren till höger på skärmen så kan du samtidigt se kod o resultat.
  • Starta Live Server med kortkommandot Alt+L Alt+O.
  • Spara med kortkommandot Ctrl+S.

Trafikljus-1

Dagnummer

if - else if - else (höger, vänster)

Logiskt och, && (kvadranter)

Villkorsoperatorn

Vanliga fel vid selektioner

Använda = istället för ==

if (x = 1) // Fel
    output("Ett")

Gör en tilldelning. Variabeln x blir 1. Hela uttrycket blir 1, vilket sedan tolkas som sant.

if (x == 1) // Rätt
    output("Ett")

Använda =< istället för <=

if (x =< 0) // Fel 
    output("Mindre än eller lika med noll")

Skriv tecknen i den ordning man säger dem; "mindre än" eller "lika med".

if (x <= 0) // Rätt 
    output("Mindre än eller lika med noll")

Inte använda else-if om flera alternativ

if (x == 0)
    output("Noll")
if (x == 1) // Fel
    output("Ett") 
else
    output("Annat")

Om x har värdet 0 så kommer både "Noll" och "Annat" att skrivas ut!

if (x == 0)
    output("Noll") 
else if (x == 1) // Rätt
    output("Ett") 
else
    output("Annat")

Använda &&, eller ||, utan två villkor

if (x == 0 || 1) // Fel
    output("Noll eller ett")

På båda sidor om && så måste det finnas ett villkor.

if (x == 0 || x == 1) // Rätt 
    output("Noll eller ett")

Samma sak gäller för &&.

Testa intervall utan &&

if (0 <= x <= 10) // Fel 
    output("I intervallet [0, 10]")

De två villkoren måste sättas ihop med &&.

if (0 <= x && x <= 10) // Rätt 
    output("I intervallet [0, 10]")

Använda indentering istället för satsblock

if (x < 0)
    output("Negativt. Byt tecken") 
    x = -x; // Fel

Den sista satsen kommer alltid att utföras, även då x är positivt. Fixa till det genom att skapa ett satsblock.

if (x < 0) {                        // Rätt
    output("Negativt. Byt tecken");
    x = -x
}

Fel ordning vid flera olikhetstester

if (n < 1000)
    output("n mindre än 1000") // Fel
else if (n < 100)
    output("n mindre än 100")
else if (n < 10)
    output("n mindre än 10")

Den första satsen blir sann för alla värden mindre än tusen, de övriga testas aldrig. Ett n-värde på 42 skulle producera utskriften "n mindre än 1000" och inte, som avsett, "n mindre än 100".

if (n < 10)
    output("n mindre än 10") // Rätt
else if (n < 100)
    output("n mindre än 100")
else if (n < 1000)
    output("n mindre än 1000")

Genomgång funktioner

Funktioner behöver ett namn som följs av parenteser. I exemplet nedan är namnet alltså 'mindre'.
Funktionen kan ta argument. I exemplet är x och y argument. Det är variabler som bara funktionen kommer åt.
När du ska köra en funktion så skriver du namnet följt av parenteser. I parenteserna kan du ange vilka värden argumenten ska ha.
För att resten av programmet ska få tillgång till något som händer i funktionen behöver det returneras. Du skriver return och sedan vilket värde som funktionen ska ge tillbaka.
I exemplet returneras true eller false. Variablerna m1, m2 och m3 blir då true eller false beroende på vad som händer i funktionen.
I exemplet finns y:number = 10 där funktionen deklareras.

Därför fungerar det att skriva mindre(2), vilket är samma som om man hade skrivit mindre(2, 10).
